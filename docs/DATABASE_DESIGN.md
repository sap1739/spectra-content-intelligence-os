# Database Design

PostgreSQL 17 (pgvector image), Prisma ORM, migration-based schema management.
Schema source of truth: `packages/database/prisma/schema.prisma`.

## 1. Conventions

- **UUID primary keys** generated by the database (`gen_random_uuid()`, pgcrypto).
- **UTC everywhere**: all timestamps are `timestamptz`; user/workspace timezones are explicit
  display-only columns (`users.timezone`, `workspaces.timezone`).
- **Tenant columns**: every tenant-owned table carries `organizationId` (+ `workspaceId`
  where workspace-scoped) with FK + composite indexes.
- **Soft deletion** via `deletedAt` on user-facing entities; `audit_logs` is append-only.
- **Audit metadata**: `createdAt`, `updatedAt`, `createdById` where meaningful.
- **JSONB** only for shapes validated by `@spectra/contracts` at the boundary (documented per
  column in the schema).
- Table names are snake_case via `@@map`.

## 2. Phase 1 entities

| Table             | Purpose                                                   | Key constraints / indexes                                                                    |
| ----------------- | --------------------------------------------------------- | -------------------------------------------------------------------------------------------- |
| users             | Identity, timezone, locale, status                        | unique(email)                                                                                |
| organizations     | Billing/admin tenant root                                 | unique(slug)                                                                                 |
| memberships       | User↔org role + extra permissions + workspace restriction | unique(organizationId, userId); index(userId)                                                |
| workspaces        | Primary isolation boundary                                | unique(organizationId, slug); index(org, status)                                             |
| brands            | Brand voice/guidelines                                    | unique(workspaceId, slug); index(workspace, status)                                          |
| custom_verticals  | User-defined niche steering research                      | unique(workspaceId, slug)                                                                    |
| research_projects | Research container (objective, vertical, brand)           | index(workspaceId, status)                                                                   |
| research_runs     | One pipeline execution: status, trigger, stage, stats     | index(projectId, status)                                                                     |
| research_sources  | Discovered/retrieved source with provenance               | unique(workspaceId, urlHash); index(projectId, processingStatus); index(duplicateClusterKey) |
| research_findings | Extracted finding w/ excerpt, scores, provenance          | index(projectId, status); index(sourceId)                                                    |
| trend_candidates  | Trend w/ state, latest explainable score                  | index(workspaceId, state); index(workspaceId, normalizedScore)                               |
| audit_logs        | Append-only action trail w/ correlationId                 | index(org, createdAt); index(resourceType, resourceId)                                       |

Deletion rules: org/workspace cascade to owned data; optional references (`verticalId`,
`brandId`, `runId`, `createdById`) use `SET NULL` so history survives entity removal.

## 3. Duplicate detection support

- `urlHash` = SHA-256 of the normalized URL, unique per workspace (exact duplicates).
- `contentHash` = SHA-256 of extracted text (near-duplicate clustering input).
- `duplicateOfSourceId` self-relation for exact duplicates; `duplicateClusterKey` groups
  near-duplicates across sources and findings.

## 4. Tenant enforcement in the data layer

1. Services authorize with `assertTenantOwnership` (same error for missing vs. foreign rows —
   guessed identifiers learn nothing).
2. The Prisma client extension (`createPrismaClient`) rejects multi-row operations on
   tenant-scoped models without an `organizationId` filter (`TenantScopeViolationError`) —
   defence in depth, unit-tested in `tenant-guard.test.ts`.
3. Row-Level Security is a deliberate Phase 2+ hardening option once the session/principal
   mechanism exists to set `app.current_org` per connection (documented, not yet enabled).

## 5. Vector search

Migration `..._init` enables the `vector` extension. Phase 2 adds embedding tables
(`document_chunks` with a `vector` column + HNSW index) behind `VectorStoreProvider`
(ADR-0005); no vector columns exist in Phase 1 tables.

## 6. Migrations & seed

- `pnpm db:migrate` (dev) / `pnpm db:deploy` (CI/prod) — plain SQL migrations in
  `packages/database/prisma/migrations`.
- `pnpm db:seed` is idempotent and deterministic (fixed UUIDs): demo user, organization,
  membership, workspace, brand, one fully worked vertical and one DRAFT research project.
  **No fabricated findings, trends or analytics.**

## 7. Future entities

Contract-only entities (claims, citations, evidence packs, strategy/content, social accounts,
knowledge documents) are documented in [DOMAIN_MODEL.md](DOMAIN_MODEL.md) §6 and materialize
